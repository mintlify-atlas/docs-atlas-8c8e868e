## Overview

The `ui/widgets.py` module provides factory functions for creating styled Tkinter widgets with consistent colors, fonts, and borders across the entire application.

**Key benefit:** Centralized styling — change the appearance of all buttons or entries by editing a single function.

Defined in: `source/ui/widgets.py`

## Global Styling

### `apply_style(root)`

Configures the global ttk theme for the application. Must be called once during initialization.

<ParamField path="root" type="tk.Tk" required>
  The Tkinter root window instance
</ParamField>

**Configures:**
- `TNotebook` — Tab container styling
- `TNotebook.Tab` — Individual tab appearance (selected/unselected states)
- `TFrame`, `TLabelframe` — Container backgrounds and borders
- `TCombobox` — Dropdown menus
- `TCheckbutton` — Checkboxes
- `TScrollbar` — Scrollbar appearance

**Theme:** Uses the "clam" ttk theme for maximum cross-platform customization.

Source: `widgets.py:20-55`

### `make_entry(parent, width=18, show="", state="normal")`

Creates a text entry field with dark styling and accent-colored focus border.

<ParamField path="parent" type="tk.Widget" required>
  Parent widget
</ParamField>

<ParamField path="width" type="int" default="18">
  Width of the entry field in characters
</ParamField>

<ParamField path="show" type="str" default="''">
  Character to display instead of actual input:
  - `''` (empty) for plain text
  - `'*'` for password masking
</ParamField>

<ParamField path="state" type="str" default="'normal'">
  Widget state: `'normal'` or `'disabled'`
</ParamField>

**Returns:** `tk.Entry`

**Styling:**
- Background: `BG3` (dark gray)
- Text color: `TEXT` (light gray/white)
- Focus highlight: `ACCENT` (blue)
- Border: `BORDER` (subtle outline)

### `make_scrolled_frame(parent)`

Creates a vertically scrollable frame using Canvas and Scrollbar. Useful for tabs with extensive vertical content.

<ParamField path="parent" type="tk.Widget" required>
  Parent widget
</ParamField>

**Returns:** `tk.Frame` — The inner frame where you should place child widgets

**Features:**
- Automatic scroll region recalculation on resize
- Mouse wheel support
- Dark theme styling

**Implementation:**
- Uses `tk.Canvas` with embedded `tk.Frame`
- Binds `<Configure>` event to update scroll region
- Binds `<MouseWheel>` for scroll interaction

Source: `widgets.py:146-175`

## Color Constants

All widget functions reference color constants from `constants.py`:

| Constant | Usage |
|----------|-------|
| `BG` | Main background (black) |
| `BG2` | Panel background (dark gray) |
| `BG3` | Input field background |
| `ACCENT` | Primary accent (blue) |
| `ACCENT2` | Action color (orange/red) |
| `SUCCESS` | Success indicators (green) |
| `TEXT` | Primary text color |
| `TEXT2` | Secondary text color |
| `BORDER` | Border/outline color |

## Example Usage

```python
from ui.widgets import make_frame, make_button, make_entry, apply_style
import tkinter as tk

root = tk.Tk()
apply_style(root)  # Apply global theme first

frame = make_frame(root)
frame.pack()

entry = make_entry(frame, width=25)
entry.pack(side='left', padx=5)

btn = make_button(frame, "Submit", lambda: print(entry.get()))
btn.pack(side='left')

root.mainloop()
```
